# spec2openapi v0.3.0

[한국어 릴리즈 노트](v0.3.0.ko.md)

A library-first release: the Python API got the same care the CLI already
had, and the Swagger upgrader was hardened against a large sweep of
real-world documents until every one of them converts clean.

## Highlights

**Proven against real-world specs.** A new opt-in corpus suite
(`python -m pytest -m corpus`) converts a stratified sample of public
Swagger 2.0 definitions from the [APIs.guru](https://apis.guru) directory
and checks every output three ways: the OpenAPI 3.0 validator, the 3.1
validator, and a live `FastMCP.from_openapi()` round-trip. The full
directory population (975 testable documents) was swept during this cycle;
every converter defect it surfaced is fixed in this release, and the
known-failures allowlist is **empty**.

**A first-class library.** If you embed spec2openapi instead of calling
the CLI:

- `import spec2openapi` no longer loads the SOAP stack — a Swagger-only
  import went from ~200 ms to ~15 ms; zeep/lxml load on first SOAP use.
- The package ships `py.typed` (PEP 561), so mypy/pyright see the
  annotations.
- `check_fastmcp_ready(spec) -> list[str]` exposes the static
  FastMCP-readiness checks behind `spec2openapi validate` as a
  dependency-free function (the CLI now consumes the same code) — it also
  predicts operationId collisions that only appear after FastMCP's
  tool-name normalization.
- Every failure path raises `ConversionError` (a `ValueError`): wrong
  `openapi_version` values are rejected loudly instead of silently
  emitting 3.0, non-UTF-8 sources fail with a labeled error instead of
  silently corrupting text, and `spec_has_soap` returns `False` on
  malformed input instead of crashing.
- The README Library section now documents error handling, programmatic
  `x-s2o` access, `strict`, `forbid_external`, and the public-API
  boundary (`__all__`).

**Swagger upgrader: the corpus wave.** Real-world documents do surprising
things; the upgrader now converts all of these to validator-clean output
instead of passing them through broken:

- Deep local `$ref`s (`#/paths/.../responses/200/schema/...`,
  `#/definitions/Foo/properties/bar`) are hoisted once per unique pointer
  into synthesized `components.schemas` entries — output stays linear
  even when hoisted targets reference each other, and reference cycles
  become legal recursive schemas.
- Dangling `$ref`s become `{}` with a lossy record; percent-encoded
  `$ref` tokens are decoded before key lookup.
- A property literally named `default`/`enum`/`example`/`discriminator`
  is treated as a property, not as the same-named keyword.
- Duplicate parameters are deduplicated by resolved (name, in);
  parameter `$ref`s without a component home are resolved and inlined.
- Component keys are sanitized across every namespace with all references
  rewritten; `type`-mismatched defaults are coerced or dropped;
  uncompilable `pattern`s and items-level `collectionFormat` are
  preserved as `x-` extensions; string-valued `consumes`/`produces` no
  longer split into characters.
- Widely-used vendor extensions are promoted to native keywords:
  parameter `x-example` → `example`, `x-oneOf`/`x-anyOf` →
  `oneOf`/`anyOf`.

**A tighter `x-s2o` report.** `--strict`/`strict=True` fails the
conversion when any assumption or lossy transformation would be needed;
missing response descriptions are filled with standard HTTP status
phrases; identical records aggregate to `<message> (×N)` instead of
repeating (one real document carried the same line 171 times); http(s)
URLs are accepted everywhere a spec path is.

## Install

```bash
pip install spec2openapi          # converter + CLI
pip install "spec2openapi[mcp]"   # + SOAP bridge (required to serve SOAP)
```

## What's Changed

- Wrap string-valued consumes/produces instead of splitting into characters (#56)
- Normalize five more non-conformant inputs to valid OpenAPI 3 (#58)
- Inline global formData parameters into the form requestBody (#60)
- Fill missing response descriptions with the HTTP status phrase (#62)
- Strict mode for upgrade and record the last silent auto-fixes (#64)
- Declare server variables for templated host/basePath (#66)
- Wrap schema $ref siblings in allOf so they survive OpenAPI 3.0 (#68)
- Accept http(s) URLs for Swagger/OpenAPI sources in load_spec (#70)
- Opt-in real-world corpus sweep (#77)
- Sanitize component keys in every namespace and rewrite references (#78)
- Decode percent-encoded $ref tokens before key mapping (#79)
- Coerce or drop schema defaults that violate their own schema (#80)
- Preserve items-level collectionFormat as x-collectionFormat (#81)
- Inline deep local $refs that would dangle after conversion (#82)
- Validate the openapi_version argument at library entry points (#89)
- Keep library entry points inside the ConversionError contract (#90)
- Ship a py.typed marker (PEP 561) (#91)
- Lazy-load the SOAP stack for Swagger-only imports (#92)
- Expose the FastMCP-readiness static checks as check_fastmcp_ready (#93)
- Document the library-first surface (#94)
- Resolve dangling simple $refs and keyword-named properties (#100)
- Dedupe parameters and hoist deep refs instead of inlining (#102)
- Preserve uncompilable schema patterns as x-pattern (#103)
- Promote x-example, x-oneOf, and x-anyOf to native keywords (#104)
- Aggregate repeated x-s2o records with counts (#105)
