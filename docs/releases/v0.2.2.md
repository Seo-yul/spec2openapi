# spec2openapi v0.2.2

[한국어 릴리즈 노트](v0.2.2.ko.md)

A small developer-experience patch: when input is malformed or the wrong
type, the error now tells you **where and how** it went wrong instead of a
cryptic traceback.

## Highlights

**Traceable format errors.** Syntax errors in a spec file now name the file
and the location:

- JSON/YAML errors are prefixed with the source path (`<file>: invalid
  JSON — … line 1 column 18`), keeping the parser's line/column.
- A malformed WSDL is pre-parsed, so you get `invalid XML in <file>: …
  line X, column Y` instead of a misleading "no convertible operations" or
  a cryptic zeep-internal message.

**Clear library input guards.** When you call the library directly (no file
involved), passing the wrong type fails fast with a clear message:

- `convert_swagger` expects a mapping and `convert_wsdl` a path/URL string;
  a mismatch raises `ConversionError` naming the expected and actual type.
- `is_swagger2` returns `False` for a non-mapping instead of crashing.

Structural validity remains the job of `spec2openapi validate` — these
changes are about *traceable* errors, not more validation.

## Install

```bash
pip install spec2openapi          # converter + CLI
pip install "spec2openapi[mcp]"   # + SOAP bridge (required to serve SOAP)
```
