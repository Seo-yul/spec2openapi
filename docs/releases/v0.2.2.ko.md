# spec2openapi v0.2.2

[English release notes](v0.2.2.md)

작은 개발자 경험(DX) 패치입니다. 입력이 깨졌거나 타입이 틀렸을 때, 이제 오류가
cryptic한 트레이스백 대신 **어디서 어떻게** 틀렸는지 알려줍니다.

## 주요 변경

**추적 가능한 형식 오류.** 스펙 파일의 문법 오류가 이제 파일과 위치를 알려줍니다.

- JSON/YAML 오류는 소스 경로를 앞에 붙입니다(`<파일>: invalid JSON — … line 1
  column 18`). 파서가 주는 줄·칸은 그대로 유지됩니다.
- 깨진 WSDL은 미리 파싱해서, 오도하는 "변환 가능한 오퍼레이션 없음"이나
  cryptic한 zeep 내부 오류 대신 `invalid XML in <파일>: … line X, column Y`로
  정확한 XML 문법 위치를 알려줍니다.

**명확한 라이브러리 입력 가드.** 파일 없이 라이브러리를 직접 호출할 때, 틀린
타입을 넘기면 곧바로 명확한 메시지로 실패합니다.

- `convert_swagger`는 mapping을, `convert_wsdl`은 경로/URL 문자열을 기대하며,
  타입이 어긋나면 기대/실제 타입을 알려주는 `ConversionError`를 발생시킵니다.
- `is_swagger2`는 non-mapping에 대해 크래시 대신 `False`를 반환합니다.

구조 정밀 검증은 여전히 `spec2openapi validate`의 몫입니다 — 이번 변경은
검증을 더 하는 게 아니라 오류를 *추적 가능*하게 만드는 것입니다.

## 설치

```bash
pip install spec2openapi          # 변환기 + CLI
pip install "spec2openapi[mcp]"   # + SOAP 브리지 (SOAP 서빙에 필수)
```

- 형식 오류를 추적 가능하게 — 파일명 + XML 위치 (#49)
- 라이브러리 진입점의 잘못된 입력 타입 가드 (#51)
