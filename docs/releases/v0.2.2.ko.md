# spec2openapi v0.2.2

[English release notes](v0.2.2.md)

개발 경험을 다듬은 작은 패치입니다. 입력이 깨졌거나 타입이 틀렸을 때, 이제
정체를 알기 어려운 트레이스백 대신 **어디가 어떻게** 틀렸는지 알려줍니다.

## 주요 변경

**형식 오류를 추적할 수 있게 됐습니다.** 스펙 파일에 문법 오류가 있으면
파일과 위치를 짚어 줍니다.

- JSON/YAML 오류에는 소스 경로가 앞에 붙습니다(`<파일>: invalid JSON — …
  line 1 column 18`). 파서가 알려주는 줄·열 정보는 그대로 유지됩니다.
- 깨진 WSDL은 미리 한 번 파싱해서 `invalid XML in <파일>: … line X,
  column Y`로 XML 문법 오류의 정확한 위치를 알려줍니다. 예전처럼 엉뚱한
  "변환 가능한 오퍼레이션 없음"이나 zeep 내부 오류가 나오지 않습니다.

**라이브러리 입력 가드가 명확해졌습니다.** 파일 없이 라이브러리를 직접
호출하다 타입을 잘못 넘기면 곧바로 명확한 메시지로 실패합니다.

- `convert_swagger`는 mapping을, `convert_wsdl`은 경로/URL 문자열을
  기대합니다. 타입이 어긋나면 기대한 타입과 실제 타입을 알려주는
  `ConversionError`를 던집니다.
- `is_swagger2`는 mapping이 아닌 값에 크래시하지 않고 `False`를 반환합니다.

구조 검증은 여전히 `spec2openapi validate`의 몫입니다. 이번 변경은 검증을
더 하는 것이 아니라 오류를 *추적 가능*하게 만드는 것입니다.

## 설치

```bash
pip install spec2openapi          # 변환기 + CLI
pip install "spec2openapi[mcp]"   # + SOAP 브리지 (SOAP 서빙에 필수)
```

## 포함된 변경

- 형식 오류를 추적 가능하게 — 파일명 + XML 위치 (#49)
- 라이브러리 진입점의 잘못된 입력 타입 가드 (#51)
