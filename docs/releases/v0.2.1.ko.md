# spec2openapi v0.2.1

[English release notes](v0.2.1.md)

**Swagger 2.0 → OpenAPI 3.x** 업그레이더의 견고성·정합성 릴리즈입니다. 실제
Swagger 문서에는 사소한 누락과 기술적으로 무효한 구조가 많은데, 이번 릴리즈는
결정적 교정이 가능한 경우 그것들을 **유효한 OpenAPI 3**으로 바꾸고, 변환이
진짜 불가능한 경우에는 **명확한 에러**를 냅니다 — 깨지거나 손실된 스펙을 조용히
내보내지 않습니다. 모든 자동 교정은 출력의 `x-s2o` 리포트에 기록됩니다.

## 주요 변경

**불완전한 입력에서도 유효한 OpenAPI 3.** 순진한 변환이 놓치는, 문맥에 따라
필수/강제되는 규칙들을 이제 지킵니다:

- `in: path` 파라미터는 `required: true`로 강제; 경로 키에 `/` 접두;
  대응 파라미터가 없는 `{template}` 세그먼트에는 파라미터를 자동 주입.
- `items` 없는 `type: array` 스키마에 `items: {}` 보충; JSON-Schema `type`
  배열은 3.0에서 단일 타입 + `nullable`로 축약.
- 무효한 `securitySchemes`(알 수 없는 type, `name`/`in` 없는 `apiKey`, 필수
  URL 없는 `oauth2` flow)는 내보내지 않고 드롭; 부분 누락된 `info`
  (`title`/`version`)는 완성.
- 공백/슬래시가 들어간 `definitions` 이름은 유효한 컴포넌트 키로 정규화하고
  모든 `$ref`도 함께 재작성.
- `allowEmptyValue`는 query 파라미터에만 유지; `collectionFormat`은 위치가
  허용하는 `style`로 매핑; `name` 없는 `tag`는 드롭.

**조용히 넘기지 않고 명확히 실패.** 소스를 충실히 변환할 수 없을 때 새
`ConversionError`(공개 export, `ValueError` 하위 클래스)를 발생시킵니다 —
타입을 풀 수 없는 XSD element, 또는 변환 가능한 SOAP 오퍼레이션이 0개인 WSDL.
CLI는 트레이스백이나 빈 스펙 대신 한 줄 `error: …`를 출력하고 exit code 2로
종료합니다.

**명시적이고 감사 가능한 출력.** 변환된 모든 파라미터는 이제 명시적 boolean
`required`를 가집니다. 모든 기본값·강제 변환·드롭·리네임은
`x-s2o.assumptions` / `x-s2o.lossy`에 기록되어, 조용히 바뀌는 것이 없습니다.

## 알아둘 동작 변화

- 출력 스펙이 0.2.0과 미세하게 다릅니다(명시적 `required`, 정규화된 경로,
  정규화된 컴포넌트 키) — 모두 유효한 OpenAPI 3이고 의미는 동일합니다.
- 예전엔 빈 스펙이나 미묘하게 무효한 스펙으로 "성공"하던 변환이 이제
  `ConversionError`를 낼 수 있습니다. 라이브러리를 스크립트로 감싸 쓴다면
  `ValueError`로 잡으면 됩니다.

## 설치

```bash
pip install spec2openapi          # 변환기 + CLI
pip install "spec2openapi[mcp]"   # + SOAP 브리지 (SOAP 서빙에 필수)
```

## 포함된 변경 (What's Changed)

- `in: path` 파라미터를 `required: true`로 강제 (#16)
- 무효한 `securitySchemes`를 내보내지 않고 드롭 (#18)
- 부분 누락된 `info` 완성 + 누락된 path 파라미터 주입 (#20)
- 파라미터 위치 규칙 준수 (`allowEmptyValue`, `collectionFormat`) (#22)
- 비-정합 입력에 대한 업그레이더 방어 강화 (#24)
- 변환 불가 입력을 조용히 넘기지 않고 `ConversionError` 발생 (#26)
- 기록 없던 안전한 기본값/skip을 기록 (#28)
- 변환된 파라미터에 항상 명시적 `required` 부여 (#30)
- 경로 `/` 접두, `name` 있는 tag, array `items` 강제 (#32)
- Swagger definition 이름을 유효한 컴포넌트 키로 정규화 (#34)
