# spec2openapi v0.2.1

[English release notes](v0.2.1.md)

**Swagger 2.0 → OpenAPI 3.x** 업그레이더를 단단하게 다듬은 릴리즈입니다.
실제 Swagger 문서에는 사소한 누락과 규격에 어긋난 구조가 흔합니다. 이번
릴리즈부터는 결정적으로 교정할 수 있는 문제라면 고쳐서 **유효한 OpenAPI 3**을
만들고, 도저히 변환할 수 없는 문서라면 깨진 스펙을 조용히 내보내는 대신
**명확한 에러**로 실패합니다. 자동으로 교정한 내용은 모두 출력의 `x-s2o`
리포트에 남습니다.

## 주요 변경

**불완전한 입력에서도 유효한 OpenAPI 3을 만듭니다.** 단순 변환으로는 놓치기
쉬운, 문맥상 반드시 지켜야 하는 규칙들을 챙깁니다.

- `in: path` 파라미터는 `required: true`로 강제하고, 경로 키에는 `/`를
  붙이며, 대응 파라미터가 없는 `{template}` 세그먼트에는 파라미터를 자동으로
  넣어 줍니다.
- `items` 없는 `type: array` 스키마에는 `items: {}`를 보충하고, JSON Schema의
  `type` 배열은 3.0에 맞게 단일 타입과 `nullable`로 바꿉니다.
- 무효한 `securitySchemes`(알 수 없는 type, `name`/`in` 없는 `apiKey`, 필수
  URL이 빠진 `oauth2` flow)는 내보내지 않고 버리며, `title`/`version`이 빠진
  `info`는 채워 넣습니다.
- 공백이나 슬래시가 들어간 `definitions` 이름은 유효한 컴포넌트 키로 바꾸고,
  이를 가리키는 `$ref`도 전부 함께 고칩니다.
- `allowEmptyValue`는 query 파라미터에만 남기고, `collectionFormat`은 위치가
  허용하는 `style`로 매핑하며, `name` 없는 `tag`는 버립니다.

**조용히 넘어가지 않고 명확하게 실패합니다.** 소스를 충실히 변환할 수 없으면
새로 공개된 `ConversionError`(`ValueError`의 하위 클래스)를 던집니다. 타입을
풀 수 없는 XSD element가 있거나, 변환 가능한 SOAP 오퍼레이션이 하나도 없는
WSDL이 여기에 해당합니다. CLI는 트레이스백이나 빈 스펙 대신 `error: …` 한
줄을 출력하고 exit code 2로 종료합니다.

**출력을 명시적이고 감사 가능하게 만들었습니다.** 변환된 모든 파라미터에
boolean `required`가 명시적으로 붙습니다. 기본값 적용, 강제 변환, 드롭,
이름 변경은 하나도 빠짐없이 `x-s2o.assumptions`와 `x-s2o.lossy`에 기록되어
조용히 바뀌는 것이 없습니다.

## 알아둘 동작 변화

- 출력 스펙이 0.2.0과 조금 다릅니다(명시적 `required`, 정규화된 경로와
  컴포넌트 키). 모두 유효한 OpenAPI 3이고 의미는 같습니다.
- 예전에는 빈 스펙이나 미묘하게 무효한 스펙을 내면서 "성공"하던 변환이 이제
  `ConversionError`로 실패할 수 있습니다. 라이브러리를 스크립트로 감싸
  쓰신다면 `ValueError`로 잡으면 됩니다.

## 설치

```bash
pip install spec2openapi          # 변환기 + CLI
pip install "spec2openapi[mcp]"   # + SOAP 브리지 (SOAP 서빙에 필수)
```

## 포함된 변경

- `in: path` 파라미터를 `required: true`로 강제 (#16)
- 무효한 `securitySchemes`를 내보내지 않고 드롭 (#18)
- 부분 누락된 `info` 완성 + 누락된 path 파라미터 주입 (#20)
- 파라미터 위치 규칙 준수 (`allowEmptyValue`, `collectionFormat`) (#22)
- 규격에 어긋난 입력에 대한 업그레이더 방어 강화 (#24)
- 변환 불가 입력을 조용히 넘기지 않고 `ConversionError` 발생 (#26)
- 기록 없던 안전한 기본값과 skip을 기록 (#28)
- 변환된 파라미터에 항상 명시적 `required` 부여 (#30)
- 경로 `/` 접두, `name` 있는 tag, array `items` 강제 (#32)
- Swagger definition 이름을 유효한 컴포넌트 키로 정규화 (#34)
