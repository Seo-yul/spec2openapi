# spec2openapi v0.4.0

[English release notes](v0.4.0.md)

WSDL 변환기 릴리즈다. substitution group을 엔드투엔드로 지원하게 됐고, WSDL
입력이 Swagger 경로만큼 유연해졌으며, 변환된 스키마에 LLM이 인자 형식을
맞추는 데 도움이 되는 example 값이 실린다.

## 하이라이트

**XSD substitution group, 엔드투엔드 지원.** substitution head 참조는
이전엔 head의 base 타입으로만 변환됐다 — 멤버 element들은 조용히 사라졌고,
abstract head는 어떤 유효 인스턴스도 만족할 수 없는 스키마를 만들었다.
이제 head 참조는 **자기서술 단일키 브랜치의 `oneOf`**가 된다:

```json
"payment": {"oneOf": [
  {"required": ["creditCard"],   "properties": {"creditCard":   {"$ref": "…"}}},
  {"required": ["bankTransfer"], "properties": {"bankTransfer": {"$ref": "…"}}}
]}
```

JSON이 와이어 element를 스스로 명명하므로 SOAP 브리지는 선택된 멤버를
그대로 직렬화하고(head는 와이어에 절대 등장하지 않음), 응답에서는 실제
element를 대칭으로 역래핑한다. 추이적 멤버십과 abstract 제외를 처리하며,
새 `x-soap-substitution` 마커가 어떤 런타임에서든 쓸 수 있도록 계약을
문서화한다. abstract head가 와이어에 나타나면 fault를 내는 목 SOAP 서버로
엔드투엔드 검증까지 마쳤다.

**Swagger만큼 유연해진 WSDL 입력.** `convert_wsdl`(과 `parse_wsdl`)은
이제 다음 중 정확히 하나를 받는다:

```python
spec2openapi.convert_wsdl("service.wsdl")          # 경로 / URL (기존)
spec2openapi.convert_wsdl("vendor-bundle.zip")     # zip 번들
spec2openapi.convert_wsdl(content=wsdl_bytes)      # 문서 본문 자체
spec2openapi.convert_wsdl(files={"service.wsdl": …, "types.xsd": …})
```

번들은 상대 `xsd:import`를 내부에서 해소하고(`forbid_external=True`에서도
오프라인 동작), 멤버 이름은 zip-slip을 차단하며, 비압축 총량은 명시적으로
상한이 걸리고, `x-soap.wsdl`에는 임시 경로 대신 논리 라벨(`<memory>`,
`<bundle:entry>`, `archive.zip!entry`)이 실린다. CLI는
`spec2openapi convert -`로 stdin을 읽는다.

**결정적 example 값.** 변환된 스키마에 원본에서 유도한 `example`이 실린다
— 첫 enumeration 값, 또는 날짜/시간/기간 format의 고정 표기 예시. 값을
합성하는 일은 없다: 맨 문자열·숫자는 example 없이 남고, XSD `default`가
있는 프로퍼티는 default만 유지한다.

**더 단단해진 에러 계약.** WSDL 파싱 실패가 `ConversionError` 계약 안에
머문다: strict 모드 스킵과 변환 가능 오퍼레이션 없음 케이스 포함, 소스
라벨 절단, 그리고 경로 자리에 문서 *본문*을 넘기면 `content=`를 안내하는
맞춤 힌트가 나온다.

## 설치

```bash
pip install spec2openapi          # 변환기 + CLI (Python 3.10+)
pip install "spec2openapi[mcp]"   # + SOAP 브리지 (SOAP 서빙에 필수)
```
