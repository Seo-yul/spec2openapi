# spec2openapi v0.3.0

[English release notes](v0.3.0.md)

라이브러리 사용성에 초점을 맞춘 릴리즈입니다. CLI 수준으로만 다듬어져 있던
파이썬 API를 같은 수준까지 끌어올렸고, 실전 문서 수백 개를 돌려 찾아낸
결함을 전부 고칠 때까지 Swagger 업그레이더를 손봤습니다.

## 하이라이트

**실전 스펙으로 검증했습니다.** 새로 추가된 opt-in 코퍼스 스위트
(`python -m pytest -m corpus`)는 공개 [APIs.guru](https://apis.guru)
디렉터리에서 Swagger 2.0 문서를 층화 표본으로 받아 변환하고, 출력마다 세
가지 검사를 통과시킵니다. OpenAPI 3.0 validator, 3.1 validator, 그리고 실제
`FastMCP.from_openapi()` 라운드트립입니다. 이번 사이클에서는 디렉터리
전량(테스트 가능한 975개)을 돌렸고, 거기서 나온 변환기 결함을 이 릴리즈에서
모두 수정했습니다. known-failures allowlist는 현재 **비어 있습니다**.

**라이브러리로 쓰기 좋아졌습니다.** CLI 대신 spec2openapi를 코드에 임베드해
쓰신다면 이런 점이 달라집니다.

- `import spec2openapi`가 더 이상 SOAP 스택을 로드하지 않습니다. Swagger만
  쓰는 임포트가 약 200ms에서 약 15ms로 줄었고, zeep/lxml은 SOAP 기능을 처음
  쓸 때 로드됩니다.
- 패키지에 `py.typed`(PEP 561)가 포함되어 mypy/pyright가 타입 힌트를
  인식합니다.
- `spec2openapi validate` 뒤에 있던 FastMCP 준비성 정적 검사를
  `check_fastmcp_ready(spec) -> list[str]` 함수로 공개했습니다(CLI도 같은
  코드를 씁니다). FastMCP가 tool 이름을 정규화한 뒤에야 드러나는 operationId
  충돌까지 미리 짚어 줍니다.
- 모든 실패 경로가 `ConversionError`(`ValueError`의 하위 클래스)를 던집니다.
  잘못된 `openapi_version` 값은 조용히 3.0 문서를 내는 대신 명확하게
  거부하고, UTF-8이 아닌 소스는 텍스트를 몰래 훼손하는 대신 파일명이 붙은
  에러로 실패하며, `spec_has_soap`은 잘못된 입력에 크래시하지 않고 `False`를
  반환합니다.
- README 라이브러리 섹션에 에러 처리, `x-s2o` 접근, `strict`,
  `forbid_external`, 공개 API 경계(`__all__`)를 정리해 두었습니다.

**Swagger 업그레이더, 코퍼스가 이끈 대규모 수정.** 실전 문서에는 예상 밖의
패턴이 가득합니다. 아래 입력들이 이제 깨진 채 통과되는 대신 validator를
통과하는 출력으로 변환됩니다.

- 문서의 임의 위치를 가리키는 깊은 로컬
  `$ref`(`#/paths/.../responses/200/schema/...`,
  `#/definitions/Foo/properties/bar`)는 고유 포인터마다 한 번씩 합성
  `components.schemas` 엔트리로 hoist합니다. hoist 대상들이 서로를 참조해도
  출력 크기는 원본에 비례하는 수준을 유지하고, 참조 순환은 합법적인 재귀
  스키마가 됩니다.
- 대상이 없는 `$ref`는 lossy 기록과 함께 `{}`로 바꾸고, 퍼센트 인코딩된
  `$ref` 토큰은 키를 찾기 전에 디코딩합니다.
- `default`/`enum`/`example`/`discriminator`라는 **이름**을 가진 프로퍼티를
  동명의 키워드로 오인하지 않고 평범한 프로퍼티로 처리합니다.
- 중복 파라미터는 해소된 (name, in) 기준으로 하나만 남기고, 컴포넌트에 둘 수
  없는 파라미터 `$ref`는 원본에서 찾아 인라인합니다.
- 컴포넌트 키를 모든 네임스페이스에서 정규화하고 참조를 전부 다시 씁니다.
  타입이 어긋난 default는 변환하거나 버리고, 컴파일되지 않는 `pattern`과
  items 안의 `collectionFormat`은 `x-` 확장으로 보존하며, 문자열로 적힌
  `consumes`/`produces`가 글자 단위로 쪼개지던 문제를 고쳤습니다.
- 널리 쓰이는 벤더 확장을 네이티브 키워드로 승격합니다: 파라미터의
  `x-example`은 `example`로, `x-oneOf`/`x-anyOf`는 `oneOf`/`anyOf`로.

**`x-s2o` 리포트가 깔끔해졌습니다.** `--strict`(또는 `strict=True`)를 켜면
가정이나 손실 변환이 하나라도 필요한 순간 변환이 실패합니다. 누락된 응답
description은 표준 HTTP 상태 문구로 채우고, 같은 레코드가 반복되면
`<메시지> (×N)`로 묶으며(어떤 실전 문서는 같은 줄을 171번 담고 있었습니다),
스펙 경로를 받는 모든 곳에서 http(s) URL을 쓸 수 있습니다.

## 설치

```bash
pip install spec2openapi          # 변환기 + CLI
pip install "spec2openapi[mcp]"   # + SOAP 브리지 (SOAP 서빙에 필수)
```

## 포함된 변경

- 문자열 consumes/produces를 글자 분해 대신 리스트로 래핑 (#56)
- 비표준 입력 5종 추가 정규화 (#58)
- 전역 formData 파라미터를 form requestBody로 인라인 (#60)
- 누락된 응답 description을 HTTP status 문구로 채움 (#62)
- upgrade strict 모드 + 마지막 무기록 자동수정 2건 기록 (#64)
- 템플릿 host/basePath의 server variables 선언 (#66)
- 스키마 $ref 옆 sibling을 allOf로 보존 (#68)
- load_spec의 http(s) URL 입력 지원 (#70)
- opt-in 실전 코퍼스 스윕 (#77)
- 전 네임스페이스 컴포넌트 키 정규화 + 참조 재작성 (#78)
- 퍼센트 인코딩 $ref 토큰 디코딩 (#79)
- 자기 스키마를 위반하는 default 변환/드롭 (#80)
- items 안의 collectionFormat을 x-collectionFormat으로 보존 (#81)
- 깊은 로컬 $ref가 dangling되지 않게 처리 (#82)
- 라이브러리 진입점의 openapi_version 인자 검증 (#89)
- 라이브러리 진입점의 ConversionError 계약 준수 (#90)
- py.typed 마커 동봉 (PEP 561) (#91)
- Swagger 전용 임포트의 SOAP 스택 lazy 로드 (#92)
- FastMCP 준비성 정적 검사를 check_fastmcp_ready로 공개 (#93)
- 라이브러리 우선 문서화 (#94)
- dangling simple $ref와 키워드 이름 프로퍼티 해결 (#100)
- 파라미터 dedupe + 깊은 ref를 인라인 대신 hoist (#102)
- 컴파일 불가 pattern을 x-pattern으로 보존 (#103)
- x-example/x-oneOf/x-anyOf 네이티브 승격 (#104)
- 반복 x-s2o 레코드 집계 (#105)
