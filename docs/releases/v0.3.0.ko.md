# spec2openapi v0.3.0

[English release notes](v0.3.0.md)

라이브러리 우선 릴리즈다. CLI가 이미 받던 수준의 다듬기를 파이썬 API에도
적용했고, 실전 문서를 대량으로 돌려 나온 결함을 전부 고칠 때까지 Swagger
업그레이더를 하드닝했다.

## 하이라이트

**실전 스펙으로 검증.** 새 opt-in 코퍼스 스위트(`python -m pytest -m
corpus`)는 공개 [APIs.guru](https://apis.guru) 디렉터리의 Swagger 2.0
문서 층화 표본을 변환해, 출력마다 세 가지 검사를 통과시킨다: OpenAPI 3.0
validator, 3.1 validator, 그리고 실제 `FastMCP.from_openapi()` 라운드트립.
이번 사이클에서 디렉터리 전량(테스트 가능한 975개)을 돌렸고, 거기서 나온
변환기 결함은 이 릴리즈에 전부 수정됐다 — known-failures allowlist는
**비어 있다**.

**1급 라이브러리.** CLI 대신 spec2openapi를 임베드한다면:

- `import spec2openapi`가 더 이상 SOAP 스택을 로드하지 않는다 — Swagger만
  쓰는 임포트가 약 200ms에서 약 15ms로 줄었고, zeep/lxml은 첫 SOAP 사용
  시점에 로드된다.
- 패키지에 `py.typed`(PEP 561)가 포함돼 mypy/pyright가 타입 힌트를
  인식한다.
- `check_fastmcp_ready(spec) -> list[str]`가 `spec2openapi validate` 뒤의
  FastMCP 준비성 정적 검사를 의존성 없는 함수로 노출한다(CLI도 같은 코드를
  사용). FastMCP의 tool 이름 정규화 이후에만 드러나는 operationId 충돌도
  정적으로 예측한다.
- 모든 실패 경로가 `ConversionError`(`ValueError`의 서브클래스)를 던진다:
  잘못된 `openapi_version` 값은 조용히 3.0을 내는 대신 명시적으로
  거부되고, UTF-8이 아닌 소스는 텍스트를 조용히 훼손하는 대신 라벨 붙은
  에러로 실패하며, `spec_has_soap`은 잘못된 입력에 크래시 대신 `False`를
  반환한다.
- README 라이브러리 섹션이 에러 처리, `x-s2o` 프로그래밍 접근, `strict`,
  `forbid_external`, 공개 API 경계(`__all__`)를 문서화한다.

**Swagger 업그레이더: 코퍼스 웨이브.** 실전 문서는 놀라운 짓을 한다.
업그레이더는 이제 아래 입력들을 깨진 채 통과시키는 대신 validator를
통과하는 출력으로 변환한다:

- 깊은 로컬 `$ref`(`#/paths/.../responses/200/schema/...`,
  `#/definitions/Foo/properties/bar`)는 유니크 포인터당 한 번씩 합성
  `components.schemas` 엔트리로 hoist된다 — hoist 대상들이 서로를
  참조해도 출력은 원본 크기에 선형으로 유지되고, 참조 순환은 합법적인
  재귀 스키마가 된다.
- dangling `$ref`는 lossy 기록과 함께 `{}`가 되고, 퍼센트 인코딩된
  `$ref` 토큰은 키 조회 전에 디코딩된다.
- `default`/`enum`/`example`/`discriminator`라는 **이름**의 프로퍼티는
  동명 키워드가 아니라 프로퍼티로 취급된다.
- 중복 파라미터는 해소된 (name, in) 기준으로 dedupe되고, 컴포넌트 홈이
  없는 파라미터 `$ref`는 resolve해 인라인한다.
- 컴포넌트 키는 모든 네임스페이스에서 정제되고 참조가 전부 재작성된다.
  타입이 어긋난 default는 코어스 또는 드롭되고, 컴파일 불가한 `pattern`과
  items 레벨 `collectionFormat`은 `x-` 확장으로 보존되며, 문자열
  `consumes`/`produces`는 더 이상 글자 단위로 쪼개지지 않는다.
- 널리 쓰이는 벤더 확장은 네이티브 키워드로 승격된다: 파라미터
  `x-example` → `example`, `x-oneOf`/`x-anyOf` → `oneOf`/`anyOf`.

**더 조밀해진 `x-s2o` 리포트.** `--strict`/`strict=True`는 가정이나 손실
변환이 필요해지는 순간 변환을 실패시킨다. 누락된 응답 description은 표준
HTTP status 문구로 채워지고, 동일한 레코드는 반복 대신 `<메시지> (×N)`로
집계되며(한 실전 문서는 같은 줄을 171번 담고 있었다), 스펙 경로를 받는
모든 곳에서 http(s) URL을 쓸 수 있다.

## 설치

```bash
pip install spec2openapi          # 변환기 + CLI
pip install "spec2openapi[mcp]"   # + SOAP 브리지 (SOAP 서빙에 필수)
```

## 포함된 변경

- 문자열 consumes/produces를 글자 분해 대신 리스트로 래핑 (#56)
- 비표준 입력 5종 추가 정규화 (#58)
- 전역 formData 파라미터를 form requestBody로 인라인 (#60)
- 누락된 응답 description을 HTTP status 문구로 채움 (#62)
- upgrade strict 모드 + 마지막 무기록 자동수정 2건 기록 (#64)
- 템플릿 host/basePath의 server variables 선언 (#66)
- 스키마 $ref 옆 sibling을 allOf로 보존 (#68)
- load_spec의 http(s) URL 입력 지원 (#70)
- opt-in 실전 코퍼스 스윕 (#77)
- 전 네임스페이스 컴포넌트 키 정제 + 참조 재작성 (#78)
- 퍼센트 인코딩 $ref 토큰 디코딩 (#79)
- 자기 스키마를 위반하는 default 코어스/드롭 (#80)
- items 레벨 collectionFormat을 x-collectionFormat으로 보존 (#81)
- dangling해질 깊은 로컬 $ref 처리 (#82)
- 라이브러리 진입점의 openapi_version 인자 검증 (#89)
- 라이브러리 진입점의 ConversionError 계약 준수 (#90)
- py.typed 마커 동봉 (PEP 561) (#91)
- Swagger 전용 임포트의 SOAP 스택 lazy 로드 (#92)
- FastMCP 준비성 정적 검사를 check_fastmcp_ready로 공개 (#93)
- 라이브러리 우선 문서화 (#94)
- dangling simple $ref와 키워드 이름 프로퍼티 해결 (#100)
- 파라미터 dedupe + 깊은 ref를 인라인 대신 hoist (#102)
- 컴파일 불가 pattern을 x-pattern으로 보존 (#103)
- x-example/x-oneOf/x-anyOf 네이티브 승격 (#104)
- 반복 x-s2o 레코드 집계 (#105)
